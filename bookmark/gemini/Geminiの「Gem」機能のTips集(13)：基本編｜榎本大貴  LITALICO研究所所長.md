## 前提

### 前提1：マークダウン（Markdown）の基礎知識

マークダウンとは、テキストに記号を加えることで見出しや強調などの装飾を表現する記法です。例えば、「\`太字\`」と書くと **太字** として表示され、「# 見出し」と書くと見出しとして表示されます。

書き方の詳細はここでは割愛しますが、例えば以下の記事などを参考にしてください。

[**Markdown記法 チートシート - Qiita** *Markdown記法のチートシートです。 記法はGitHub Flavored Markdownに準拠し、一部拡張していま* *qiita.com*](https://qiita.com/Qiita/items/c686397e4a0f4f11683d)

マークダウンを使ったプロンプトのほうがAIにとってわかりやすく、出力の精度が良くなります。重要なのは、 **人間にもAIにも読みやすいように書く** ということ。当たり前のようですが、意外と見落としがちです。

**半角で書く** ということが大事です。全角スペースが混入すると、AIが意図を読み取れないことがあります。

- OKな例：#, -, \*
- NGな例：＃, ー, ＊

### 前提2：Googleドキュメントでの挙動

Googleドキュメントで下書きする場合は、マークダウンでコピー・貼り付けをすると、そのままGemに移植しやすくなります。

- 通常の「コピー（Ctrl+C）」と「ペースト（Ctrl+V）」
	- 👉️テキストだけがコピーされ、ペーストされる
- 「マークダウンとしてコピー」と「マークダウンから貼り付け」
	- 👉️テキスト＋マークダウンの装飾（#とか\*とか）もセットでコピーされ、マークダウンの装飾ごとペーストされる

### 前提3：Gemの基礎知識

Gemは、 **プロンプトを保存して繰り返し使ったり、他者と共有したりできる機能** です。

カスタム指示、説明、知識を組み合わせて、再利用可能なプロンプトを作成できます。

Gemを作成する際は、以下の要素を設定します：

- **タイトル** ：Gemのタイトル。自分や他の人にわかりやすいように。
- **カスタム指示** ：Gemの核となるプロンプト部分。ここにメインの指示を書きます。
- **説明** ：Gemの用途や使い方を説明。共有する際に、他の人が理解しやすくなるので重要です。なくてもOK。
- **デフォルトツール** ：通常のテキストプロンプトに加えて「ツール」を強制的に指定できます。最近追加された新しい設定項目。
- **知識** ：参照可能なファイルやテキストを添付できます。フォーマット例やテンプレートを保存しておくと便利。なくてもOK。

## 基本編

まずは基本編から。  
ここで紹介する内容は、Gemのみならず他の場面でも基本となるものだと思います。

### TIPS1：基本構造を守る

Gemのプロンプトは以下の要素で構成すると効果的です：

1. **役割** ：AIに何の役割を担わせるか。AIが自分を何だと思って動けばいいか、最初に伝えることが重要です。
2. **目的** ：何を達成したいか。具体的に。
3. **手順** ：どのように処理するか。順序が重要な場合は番号付きリスト（1., 2., 3.）を用いる。
4. **制約事項・条件** ：守るべきルールや制限。極力箇条書きで。
5. **入力データ** ：ユーザーがGemに入力するデータを説明。ファイルの形式（PDFファイルだよ、とか）もあると良い。
6. **テンプレート** ：出力する際のテンプレート。あれば入れる。
7. **出力の見本** ：理想的なアウトプット例。見本の有無で精度が変わる。

```
あなたは[役割]です。
[目的]を達成するために、以下の手順で処理してください：

# 手順
1. [手順1]
2. [手順2]
3. [手順3]

# 制約事項
- [制約1]
- [制約2]
- 
  
# 入力データ
例）[文字起こしデータの一部]
例）[スプレッドシートの形式の説明]

# テンプレート
例）[議事録や記録のテンプレート]

# 見本
...
```

特に、意外と忘れがちなのが、 **見本** 。  
大事なことなので繰り返し書きますが、 **見本の有無で精度が変わります。**

---

### TIPS2：強調する

**強調したいこと** をアスタリスク2つで囲って強調します。重要な指示は必ず強調しましょう。

AIは強調された部分を優先的に処理する傾向があります。

```
- **この指示は必ず実行してください**
```

---

### TIPS3：リストを使う

- 順序が重要な場合は番号付きリスト（1., 2., 3.）を使います
- 順序が重要でない場合は箇条書き（- または \*）を使います

この使い分けが意外と重要で、AIが処理の順序を理解するのに役立ちます。

```python
# 条件
- ああ
- いい
  
  
# 手順
1. ああ
2. いい
```

---

### TIPS4：固定文章を出力する

Gemの出力で、固定のメッセージを表示させたい場合もありますよね。そんな時は引用の記号（>）を用いつつ、必ずそのメッセージを表示させるように明示的に指示します。

```
**必ず以下のメッセージを表示してください。**
> こんにちは。就労移行支援事業所での支援や関係機関営業などの経験が豊富で様々なプログラム開発や資料作成を行ってきました。ここではあなたのプログラム作りをサポートします。
> ...
```

---

### TIPS5：余計な出力を出させない

AIが余計な説明をしてしまうことがありますよね。そういうときには：

```
前置き、説明文、挨拶文は一切含めない。
「了解いたしました」やユーザーの指示等の説明は一切含めない。
```

---

### TIPS6：人間にYes/Noで回答できる質問をする

「固定文章を出力する」の実践例です。  
出力した内容に対して、ユーザーに確認を取らせたい場合がありますよね。そんな時は、こんな感じで書きます。

```
**以下の確認メッセージを必ず表示**してください。
> 上記の内容でよろしければ「 **OK** 」と入力してください。
> OKでない場合は、修正点を教えてください。
```

これで、AIが必ず確認メッセージを表示してくれます。  
僕も、重要な処理の前に確認を取るようにしているGemがいくつかあります。

---

### TIPS7：人間に選択肢を提示して選択させる

ユーザーに選択肢から選んでもらいたい場合もあります。

```
**以下のいずれかを選択してください**：
- [A] オプションAの説明
- [B] オプションBの説明
- [C] オプションCの説明

選択肢（A/B/C）を入力してください。
```

これは「固定文章を出力する」のTIPSとセットで使います。

---

### TIPS8：人間がコピペしやすくする

出力した内容をそのままコピペして使いたい場合、プレーンテキストで出力させると便利です：

```javascript
出力は必ず以下の形式で行ってください：

\`\`\`plaintext
[出力内容]
\`\`\`
```

これで、出力した内容をそのまま他のツールに貼り付けられます。

コピーボタンが表示されるので、出力の中に複数のコードブロックを入れることや、マークダウンを保持したままコピーすることができます。

![画像](https://assets.st-note.com/img/1767081772-XUFZ7MET6ID0dzbkSg1oRwe3.png?width=1200)

コピーしてくれるボタンが表示される

---

### TIPS9：必須項目のチェック

入力が不足している場合に、ユーザーに入力を促したいこともありますよね。そんな時は：

```
以下の必須項目がすべて入力されているか確認してください：
- 項目1 
- 項目2
- 項目3

不足している項目がある場合は、不足項目をリストアップしてユーザーに入力を促してください。
```

これで、必須項目が不足している場合に、AIが適切に入力を促してくれます。ユーザー体験がかなり良くなりますよ。

---

### TIPS10：変数の表記

変数部分は **{変数名}や\[変数名\]** で表記します。例えば、{ユーザー名}や\[日付\]のように。  
これで、AIが「ここは変数だな」と認識しやすくなります。その後の処理で変数を使って出力するように指示します。

変数とは、変わる値を指定する箱のようなもの。  
{ユーザー名}という箱には、鈴木、佐藤、田中、などが入ります。  
{日付}という箱には、2025/12/21、2025/12/24、などが入ります。  
{メール件名}という箱には、ご挨拶(LITALICO榎本)、次回のお打ち合わせの件、などが入ります。

AIが変数を認識してくれると、テンプレートに当てはめるような出力を出したいときに便利です。

```
# 出力テンプレート
[メール件名]
[本文]

株式会社ABC
榎本大貴
aaa@aaa.aaa
```
